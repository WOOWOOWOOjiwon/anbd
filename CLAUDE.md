이 파일은 이 폴더에서 작업할 때 Claude가 따라야 하는 안내서입니다.

## 이 프로젝트는

- **개발 연습용** 프로젝트입니다. (당근마켓 같은 중고거래 마켓 "아나바다")
- 기술 스택: **Next.js**(웹사이트 틀) + **Supabase**(회원/데이터 보관) + **Tailwind CSS**(디자인)
- 디자인 컨셉: 귀여운 **픽셀 아트** 스타일
- 배포: **GitHub에 올리면(push) 자동으로 Vercel이 웹사이트로 배포**합니다. (이미 연결되어 있음)
  - 배포 주소: https://anbd-chi.vercel.app

## 일하는 방식 (가장 중요)

1. **비개발자에게 맞춰 쉽게 설명하기**
   - 개발 용어가 나오면 항상 쉬운 말로 풀어서 설명합니다.
   - 예: "커밋/푸시" → "변경한 내용을 저장하고 GitHub에 올리는 것"
   - 예: "환경변수" → "비밀번호처럼 코드에 직접 안 쓰고 따로 보관하는 설정값"

2. **지금 무엇을 하는지 자주 알려주기**
   - 작업 중간중간 "지금 ~를 하고 있어요", "이제 ~를 할게요"처럼 진행 상황을 자주 말합니다.

3. **사용자가 시킨 것만, 단계적으로 하기**
   - 시키지 않은 작업을 미리 앞서가서 하지 않습니다.
   - 한 번에 하나씩, 사용자가 따라올 수 있게 단계적으로 진행합니다.
   - 다음 단계를 제안할 수는 있지만, **허락 없이 진행하지 않습니다.**

4. **할 수 있는 건 최대한 직접 하기**
   - 코드 작성, 설치, 테스트, GitHub 올리기, Vercel 설정 등 Claude가 할 수 있는 일은 직접 처리합니다.
   - **사람이 꼭 해야 하는 일만** 골라서, 따라하기 쉽게 단계별로 안내합니다.
     (예: 외부 사이트 로그인, 본인 인증, 비밀 토큰 발급 등)

5. **개발이 끝나면 바로 확인할 수 있게 열어주기**
   - 작업을 마치면 결과를 바로 눈으로 볼 수 있도록 합니다.
   - 로컬 미리보기 주소(http://localhost:3000)를 열거나, 배포 주소를 안내합니다.

## 자주 쓰는 명령어 메모 (참고용)

이 컴퓨터는 Node.js를 nvm으로 설치해서 씁니다. 터미널 명령 전에 아래를 먼저 실행해야 `npm`이 동작합니다.

```bash
export NVM_DIR="$HOME/.nvm"; [ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"; nvm use --lts
```

- 개발 서버 켜기(미리보기): `npm run dev` → http://localhost:3000
- GitHub에 올리기: 변경사항을 commit 후 `git push` (→ Vercel 자동 배포)
- `gh` CLI는 `~/bin/gh`에 있습니다.

## 현재까지 만든 기능

- 회원가입 / 로그인 / 로그아웃 (이메일 인증 포함)
- 픽셀 아트 기본 디자인
- 판매글 CRUD: 목록 보기 / 자세히 보기 / 글쓰기 / 수정 / 삭제
  - 누구나 글을 볼 수 있고, 수정·삭제는 글쓴이만 가능 (Supabase RLS 보안 적용)
  - 사진 첨부: 글(텍스트)은 DB(`products` 표)에, 사진(파일)은 Storage(`product-images` 버킷)에 분리 보관
    (한 글당 최대 5장, DB에는 사진 위치 `image_paths`만 저장)
- 좋아요(찜) / 댓글
  - 좋아요: `product_likes` 표(한 사람이 한 글에 한 번), 누르면 토글
  - 댓글: `product_comments` 표, 자기 댓글만 삭제 가능 (RLS 적용)
  - 목록 카드에 좋아요·댓글 수 표시
- 디자인: "거지맵" 컨셉 밈 문구 + 직접 그린 픽셀 이모지 세트(`PixelEmoji`: 엽전·돈·라면·불·하트·말풍선)
